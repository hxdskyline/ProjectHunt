# Demo 2.0 详细设计

## 1. 文档目标

这份文档只做一件事：

- 把 `demo2.0流程.txt` 里的六关主线，细化成可以直接照着实现的关卡与演出脚本。

这份文档不解释技术实现方法，只定义：

- 每关战斗前后发生什么
- 哪些角色在场
- 角色走到哪里
- 哪句对白什么时候说
- 哪个 UI 在什么时候出现、移动、强调、消失
- 玩家什么时候可以点击，点击后进入什么状态

---

## 2. 全局固定规则

### 2.1 画面与战斗

- 2D 横版像素自动战斗
- 玩家不手动移动
- 玩家不释放主动技能
- 玩家只在战斗后做选择

### 2.2 演出镜头规则

- 默认镜头为固定横版战斗镜头
- 重要演出时，不切复杂镜头，只做以下三种变化：
- 轻微镜头停顿
- 屏幕中央聚焦 UI
- 场景内对象高亮

### 2.3 玩家可交互时机

玩家全流程只允许三类主动操作：

1. 点击地面掉落物
2. 在三选一界面点击某个角色按钮
3. 在“发现新兵种”界面点击“返回”或“确认”

其余阶段都属于不可操作演出。

### 2.4 角色站位规则

战斗中默认站位：

- 前排：剑士
- 中排：刺客
- 后排：弓手

如果第四关法师加入并替换队员：

- 法师默认站后排
- 被替换下场的角色完全离开当前关卡战斗队伍

### 2.5 三选一界面统一布局

屏幕布局固定：

- 顶部中间：标题
- 中部偏左、中、偏右：三个角色卡片
- 每个卡片包含：
- 角色名
- 当前形态立绘
- 本次获得物品后的结果说明
- 一个立即生效按钮

视觉强调规则：

- 标题始终是当前屏最强文字
- 当前掉落物的图标只在标题下方或新兵种演出里出现，不长期悬挂
- 按钮统一用暖橙色，避免出现多种主按钮颜色

### 2.6 发现新兵种界面统一布局

固定结构：

- 中央黑底演出面板
- 左侧：当前角色
- 右侧：当前掉落物
- 中央：合成后的新兵种
- 底部左右：返回 / 确认

固定演出顺序：

1. 隐藏三选一界面
2. 原角色与物品分别出现
3. 二者飞向中心
4. 合成结果发光出现
5. 播放一次攻击展示
6. 显示返回 / 确认

### 2.7 现有效果复用原则

当前项目里已经被确认有效、并且应作为后续模板沿用的内容有：

- 第一关流星锤拾取演出
- 第一关三选一界面
- 第一关发现新兵种演出

从第二关开始，所有新物品与新兵种演出都默认按这个原则处理：

- 尽量不改第一关已经做好的美术效果与交互节奏
- 酒杯、钥匙、法师相关演出，优先套用锤子版的已有结构
- 只替换物品图标、标题、文案、角色结果和少量必要动作
- 不重新设计另一套完全不同的 UI 动线

换句话说，后续内容应该看起来像“同一套演出系统的不同物品版本”，而不是“每关一套新界面”。

### 2.8 对白框统一规则

对白统一使用同一种场景内对白框：

- 位置：屏幕下方偏中
- 结构：角色名 + 一行到两行对白
- 背景：深色半透明底
- 文字颜色：白字
- 角色名颜色：暖白或浅金

出场规则：

1. 角色停稳后，对白框淡入
2. 一句对白说完后，直接切下一句，不反复闪烁进出
3. 一段对白全部结束后，对白框整体淡出

强调原则：

- 对白框不能压过场景中央的关键物品或合成结果
- 如果中央正在播“获得神器”或“发现新兵种”，则不同时显示对白框

### 2.9 UI 动线统一规则

从第二关开始，所有新增 UI 仍服从第一关已有动线。

固定动线：

1. 战斗 UI
2. 掉落或剧情收束
3. 物品展示或剧情对白
4. 三选一或功能选择
5. 发现新兵种演出
6. 下一关

禁止行为：

- 在同一时刻叠两层主 UI
- 一边让玩家点击，一边播关键对白
- 一边展示三选一，一边保留上一段掉落提示文字

### 2.10 技能设计引用规则

主线中所有已经出现的职业 / 物品组合，如果需要写技能表现、结果名称、三选一文案方向，统一以 `武器组合技能.txt` 为准。

执行原则：

- 详细设计文档不再把主线组合写成“待补”
- 如果流程文档里保留了早期草稿说法，实际设计与实现优先服从 `武器组合技能.txt`
- 只有当 `武器组合技能.txt` 本身没有定义该主线组合时，才允许另行补充

## 3. 六关总流程

### 3.1 主线顺序

1. 第一关：抡锤哥布林
2. 第二关：燃烧村庄
3. 第三关：酒鬼食人魔
4. 第四关：冰原救法师
5. 第五关：冰骑士
6. 第六关：最终验证

### 3.2 当前高潮节点

- 高潮①：第一关流星锤分配
- 高潮②：第三关酒杯分配
- 高潮③：第五关钥匙分配

## 4. 第一关：抡锤哥布林

### 4.1 关卡目的

- 教玩家理解“打 Boss、抢物品、分配、兵种改变”的第一层循环

### 4.2 开场演出

场上角色：

- 剑士、刺客、弓手在屏幕左侧外
- 抡锤哥布林 Boss 在屏幕右侧外

角色移动：

1. 剑士先入场，走到前排站位
2. 刺客第二个入场，走到中排站位
3. 弓手第三个入场，走到后排站位
4. 哥布林 Boss 最后从右侧走入场，走到 Boss 站位

动画要求：

- 所有单位走路时必须播放移动动作
- 站定后切回待机或战斗待机

UI：

- 玩家三人血条在角色头顶常驻
- Boss 血条在屏幕上方常驻

### 4.3 战斗演出重点

- 强调哥布林手上的流星锤
- 强调大范围砸地与抡锤威胁
- 玩家三人基础攻击形成对照：
- 剑士：普通近战
- 刺客：平抛投掷
- 弓手：抛物线射击

### 4.4 Boss 死亡到掉落

顺序固定：

1. Boss 血量归零
2. Boss 完整播放死亡动画
3. Boss 死亡动画结束后销毁
4. 流星锤从原 Boss 身位上方抛出
5. 流星锤旋转落地
6. 落地后出现持续上下浮动的指示箭头
7. 屏幕上方出现提示文字：`点击拾取“哥布林流星锤”`

UI强调：

- 这一阶段全场唯一需要玩家看的 UI 是掉落物和拾取提示
- Boss 血条可隐藏

### 4.5 拾取演出

玩家点击后：

1. 地上的流星锤停止被箭头指引
2. 流星锤从地面移动到屏幕中央
3. 场景稍微压暗
4. 屏幕中央出现黑底物品框
5. 物品框外框发暖金色光
6. 中央文字出现：`获得神器：流星锤`
7. 停留约 1 秒
8. 物品展示消失
9. 进入三选一界面

### 4.6 三选一界面

标题：

- `把流星锤交给谁？`

三个角色卡内容：

- 剑士
- 效果文案：提高伤害，并有概率格挡
- 按钮：交给剑士

- 刺客
- 效果文案：攻击有概率击晕
- 按钮：交给刺客

- 弓手
- 效果文案：变为范围伤害
- 按钮：交给弓手

交互：

- 点击任意一个按钮立即进入“发现新兵种”
- 不做额外确认层

### 4.7 发现新兵种演出

标题：

- `发现新兵种`

演出顺序：

1. 左侧出现“当前角色”
2. 右侧出现“流星锤”
3. 二者同时飞向中央
4. 中央出现新兵种立绘并发暖金光
5. 播放一次新兵种攻击展示
6. 底部出现：
- 左：返回
- 右：确认

攻击展示要求：

- 锤剑士：一次明显更重的近战攻击
- 锤刺客：两段攻击，第二段体现锤
- 锤弓手：抛出流星锤并体现范围伤害感

关卡结束：

- 点击确认后进入第二关

## 5. 第二关：燃烧村庄

### 5.1 关卡目的

- 不掉 Boss 物品
- 只做铺垫与验证
- 让玩家带着第一关改造后的兵种，打一段风格明确的过渡关

### 5.2 开场演出

场景氛围：

- 背景比第一关更亮、更偏橙红
- 屏幕右侧有燃烧感元素

场上角色：

- 当前三人队伍从左侧入场
- 敌方不是单一 Boss，而是一波波燃烧小怪

角色移动：

1. 玩家三人按正常队形走入场
2. 第一组燃烧小怪从右侧走入

### 5.3 战斗结构

这一关不做复杂剧情打断，只做三件事：

1. 展示燃烧主题小怪
2. 验证第一关新兵种在新环境下继续可用
3. 在关底引出铁匠

### 5.4 关底收束演出

战斗结束后：

1. 最后一只小怪死亡
2. 玩家三人保持站位，不立刻退场
3. 铁匠从屏幕右侧偏后方走入
4. 铁匠停在玩家队伍右前方

对白顺序：

- 铁匠：`这村子，被一个爱喝酒的怪物烧了。`
- 铁匠：`那家伙往北边去了。`
- 铁匠：`如果你们还要追，我跟你们一起去。`

UI：

- 不出现三选一
- 不出现掉落
- 屏幕中央短暂停留对白后，直接进入第三关

### 5.5 第二关时序与控制

建议节奏：

- 战斗结束到铁匠出场：0.5 秒
- 铁匠三句对白总时长：6 到 8 秒
- 对白结束到切关：1 秒

逐拍要求：

1. 最后一只小怪死亡后，先让场上伤害数字与受击特效自然消失
2. 玩家三人停止攻击，但保持朝右站位
3. 铁匠从右侧偏后位置走入
4. 铁匠走到位后再开始第一句对白
5. 每句对白之间停顿 0.4 到 0.8 秒
6. 最后一句说完后停 1 秒再切关

此段 UI 重点：

- 不允许任何“奖励感 UI”提前出现
- 重点只有铁匠和对白

玩家控制：

- 全段不可点击

## 6. 第三关：酒鬼食人魔

### 6.1 关卡目的

- 引入第二件 Boss 物品：酒杯
- 建立“掉落物不一定是传统武器”的认知

### 6.2 开场演出

场上角色：

- 玩家当前三人从左侧入场
- 铁匠跟在玩家队伍后方，但不参战
- 酒鬼食人魔从右侧入场

角色站位：

- 玩家三人仍按战斗站位
- 铁匠站在玩家队伍后方更靠左的位置
- 食人魔站在 Boss 位

视觉重点：

- 左手鸡腿
- 右手木酒杯
- 走路要有醉态

### 6.3 战斗演出重点

- 食人魔要有喝酒动作
- 喷火动作要明显
- 战斗气质和第一关不同，要更荒诞、更失衡

### 6.4 Boss 死亡后的特殊过场

这是整条主线里第一段带对白互动的掉落演出，节奏必须固定。

顺序：

1. 食人魔血量归零
2. 完整播放死亡动画
3. 食人魔销毁
4. 大酒杯掉落到地面
5. 玩家准备拾取，但先不允许点击
6. 铁匠向前走两步，站到酒杯左侧

对白和动作逐拍：

1. 铁匠：`等一下。`
2. 铁匠弯腰，把酒杯捡起
3. 铁匠把酒杯翻过来看看底部
4. 铁匠用手拍一拍杯底
5. 铁匠短暂停顿后笑一下
6. 铁匠：`哈哈哈哈。`
7. 铁匠：`这食人魔，一直拿这东西喝酒。`
8. 铁匠：`你们可别学他。`
9. 铁匠把酒杯递向屏幕左侧玩家方向
10. 酒杯从铁匠手里飞到屏幕中央

演出重点：

- 这里不需要玩家点击地面酒杯
- 重点是铁匠先“鉴定”这个物品
- 铁匠在这一关第一次明确参与掉落解释

角色位置要求：

- 铁匠检查酒杯时，必须站在酒杯与玩家队伍之间
- 玩家三人保持左侧原地站位，不向前围上来
- 酒杯被铁匠拿起后，视觉中心从地面掉落物切换到铁匠手部

### 6.5 酒杯获得演出

在铁匠把酒杯递出后：

1. 屏幕压暗
2. 中央黑底物品框出现
3. 酒杯图标居中显示
4. 暖金发光
5. 文字出现：`获得神器：酒神圣杯`
6. 停留约 1 秒
7. 进入三选一

动态要求：

- 酒杯从铁匠手中飞向中央时，不瞬移
- 飞行动线沿用流星锤拾取演出的既有节奏
- 物品框、黑底、发光样式沿用第一关，不另起一套风格

### 6.6 三选一界面

标题：

- `把酒杯交给谁？`

卡片展示规则：

- 如果某个角色已经是锤系，卡面必须显示锤系当前形态
- 结果说明展示“这次拿酒杯后会变成什么”

当前确定文案：

- 剑士：酒杯套头，偶尔引起钟鸣，降低周围敌人的伤害
- 刺客：摔破圣杯当匕首
- 弓手：投石车

如果当前角色已经带锤：

- 所有锤系 + 酒杯的结果描述，以 `武器组合技能.txt` 为唯一技能口径
- 当前演出重点只负责把“锤系当前形态 + 酒杯新理解”表现清楚

UI强调对象：

- 先强调标题
- 再同时显示三张卡
- 不做复杂的逐卡出现节奏

按钮规则：

- 按钮常态与第一关一致
- 不新增额外选中态
- 点击后立即进入发现新兵种

推荐按钮文案：

- 交给剑士
- 交给刺客
- 交给弓手

### 6.7 发现新兵种演出

和第一关同结构，但要更强：

- 标题字号稍大
- 物品发光更明显
- 攻击展示的时间稍长一点

当前重点展示：

- 普通职业拿酒杯后的荒诞感
- 锤弓手拿酒杯后的“投石车感”

时序要求：

1. 三选一隐藏：0.1 秒
2. 原角色与酒杯出现：0.2 秒
3. 飞向中心并合成：0.45 到 0.6 秒
4. 合成结果发光停留：0.35 秒
5. 攻击展示：0.8 到 1.2 秒
6. 返回 / 确认按钮出现

展示建议：

- 酒杯剑士：先展示套头轮廓，再展示钟鸣与减伤感
- 酒杯刺客：先展示碎杯匕首，再展示危险的贴身攻击
- 酒杯弓手：先展示投石车器械感，再展示重型投射物
- 锤系 + 酒杯：具体结果仍以 `武器组合技能.txt` 为准，演出上优先表现“二次理解”

关卡结束：

- 点击确认后进入第四关

## 7. 第四关：冰原救法师

### 7.1 关卡目的

- 引入第四名可用角色：法师
- 第一次出现“换人”
- 第一次出现“旧物品可以给新职业”

### 7.2 开场演出

场景氛围：

- 冷色、冰原、低饱和

场上角色：

- 玩家当前三人从左侧入场
- 铁匠不跟进战场中央，只作为剧情角色在战斗后出现

### 7.3 战斗结构

1. 玩家三人与冰怪战斗
2. 关底区域出现被困的法师
3. 最后一波敌人清掉后触发救援演出

### 7.4 救法师演出

顺序：

1. 最后一只冰怪死亡
2. 镜头停顿
3. 屏幕右侧冰封区域破开
4. 法师从右侧偏后位置走出
5. 法师停在玩家队伍右前方

对白顺序：

- 法师：`总算有人来了。`
- 法师：`你们带着的那两样东西，我已经感觉到了。`
- 法师：`锤子，酒杯。它们不只是战利品。`

角色调度：

- 法师出现前，玩家三人不退场
- 法师从右侧偏后位置走出，不能像瞬间生成在中间
- 法师停位后与玩家保持清晰对话距离

UI：

- 不出现 Boss 掉落 UI
- 不出现“获得神器”
- 这一段是剧情加入，不是奖励确认

### 7.5 换人界面

这一关不是三选一，而是“替换队友”界面。

标题：

- `法师加入，替换谁下场？`

界面结构：

- 左侧三个当前上阵角色卡
- 中央一张“法师卡”
- 底部说明：`当前只能上阵三人`

交互：

- 点击任意一个当前角色卡，表示该角色下场
- 法师立即加入队伍
- 然后进入“把哪件物品交给法师”的第二层选择

界面细化：

- 标题固定在顶部
- 三张当前队友卡横排
- 法师卡单独放在上方中间或右上，不与三张旧卡混成四选一
- 被点击的旧角色卡立刻淡出
- 法师卡补入队列

### 7.6 法师物品分配界面

标题：

- `把哪件物品交给法师？`

当前只展示玩家已拥有的物品：

- 如果只有流星锤：只显示流星锤
- 如果已有流星锤和酒杯：同时显示两件

交互：

- 点击流星锤：进入锤法师演出
- 点击酒杯：如果设计允许，进入酒法师或锤酒法师路径

界面重点：

- 这里的主语不是“把物品交给谁”，而是“法师接触哪件物品”
- 整体仍复用三选一的卡片、黑底、按钮风格

建议标题：

- `让法师接触哪件物品？`

推荐按钮文案：

- 接触流星锤
- 接触酒杯

### 7.7 法师展示演出

目标：

- 强调“法师对 Boss 物品的理解和前三职业不同”

表现方式：

- 左侧法师
- 右侧所选物品
- 合成后中央出现：
- 法师
- 锤法师
- 锤酒法师

攻击展示重点：

- 锤法师：召唤巨型光锤砸下
- 锤酒法师：更偏仪式型法术，而不是直接挥武器

返回 / 确认规则：

- 返回：退回法师物品分配界面
- 确认：锁定法师路线并进入第五关

结果名建议：

- 法师
- 锤法师
- 锤酒法师

关卡结束：

- 点击确认后进入第五关

## 8. 第五关：冰骑士

### 8.1 关卡目的

- 引入第三件 Boss 物品：钥匙
- 开始验证“锤 / 酒杯 / 钥匙”的两两组合

### 8.2 开场演出

场上角色：

- 当前上阵三人从左侧入场
- 冰骑士从右侧入场

视觉重点：

- 冰骑士手里拿的是“看起来很像重武器的巨大钥匙”
- 第一眼让玩家以为那是剑
- 第二眼才看清是钥匙

### 8.3 战斗演出重点

Boss 攻击围绕钥匙：

- 插地
- 转钥匙
- 开冰门

观感要求：

- 比第一关更重
- 比第三关更冷、更秩序化
- 强调“机关感”“门锁感”

场上视觉优先级：

1. 巨大钥匙本体
2. 冰骑士的开门 / 转钥匙动作
3. 玩家当前改造兵种的输出方式

### 8.4 Boss 死亡到掉落

顺序：

1. Boss 血量归零
2. 完整播放死亡动画
3. Boss 销毁
4. 巨大钥匙掉落
5. 地面出现指示箭头
6. 屏幕上方出现提示：`点击拾取“巨大钥匙”`

动态要求：

- 钥匙掉落时比流星锤更重
- 落地后的旋转更少
- 最后要有明显“砸地定住”的感觉

### 8.5 获得演出

点击后：

1. 钥匙飞到屏幕中央
2. 黑底展示框出现
3. 外框发暖金光
4. 文字出现：`获得神器：巨大钥匙`
5. 进入三选一

### 8.6 三选一界面

标题：

- `把巨大钥匙交给谁？`

这一关的核心不是单次组合，而是“两两组合可读”。

卡片展示规则：

- 如果角色已带锤，要显示锤系当前形态
- 如果角色已带酒杯，也要显示酒杯系当前形态
- 本次结果只展示“再拿钥匙后”的新结果

重点强调对象：

- 所有已经被强化过的角色卡，都要让玩家一眼看出“这是旧兵种升级，不是新角色”

UI细节：

- 如有必要，可在角色名下加一行短标签说明当前形态
- 但标签不能压过本次钥匙组合的主效果文案

当前建议文案：

- 剑士：发条剑士
- 弓手：抛物线射出旋转的钥匙，像钻头一样
- 刺客：把钥匙当回旋镖扔
- 法师：举起钥匙，传送门自然打开，召唤射线攻击

如果角色已有旧物品：

- 锤 + 钥匙：具体技能口径查 `武器组合技能.txt`
- 酒杯 + 钥匙：具体技能口径查 `武器组合技能.txt`

### 8.7 发现新兵种演出

这是第三个高潮点，演出比前两次更长一点。

建议顺序：

1. 当前角色出现
2. 巨大钥匙出现
3. 二者飞向中心
4. 合成结果出现并有更明显的暖金爆闪
5. 攻击展示
6. 返回 / 确认

攻击展示重点：

- 强调两两组合后的系统感
- 不只是“动作变了”，而是“兵种理解变了”

高潮③附加要求：

- 这一段比前两次多停留 0.2 到 0.4 秒
- 合成瞬间的发光最强
- 攻击展示强调“最终体系的一部分”，不是单次 gimmick

推荐展示顺序：

- 剑士路线：先展示外形变化，再展示重击
- 弓手路线：先展示抛物线与旋转钥匙，再展示钻头般命中感
- 刺客路线：先展示回旋镖飞出，再展示回返轨迹
- 法师路线：先展示举钥动作，再展示传送门自然打开与射线攻击

关卡结束：

- 点击确认后进入第六关

## 9. 第六关：最终验证

### 9.1 关卡目的

- 不再引入新物品
- 只验证前五关做出的组合，是否真的让玩家想继续搭配

### 9.2 开场演出

- 当前最终上阵三人从左侧入场
- 敌方作为验证用最终战斗阵容从右侧入场

### 9.3 战斗规则

- 这一关不做复杂剧情打断
- 重点是让玩家持续看最终队伍输出
- 让不同组合的差异被完整看见

节奏要求：

- 开场不做长对白
- 尽快进入验证战斗
- 战斗结束后再一次性进入结尾镜头

UI强调对象：

- 不强调新奖励
- 重点让头顶血条、投射物、范围伤害与控制效果自然被看清

### 9.4 结束条件

1. 最终敌方全灭
2. 玩家角色停在场上
3. 屏幕停顿
4. 淡出到结尾镜头

玩家控制：

- 全关不再出现新的 Build 选择
- 战斗结束后不弹三选一

## 10. 结尾镜头

### 10.1 场景结构

- 回到村庄
- 铁匠炉重新点火
- 玩家队伍站在屏幕左侧偏后
- 铁匠站在工作台前

### 10.2 演出顺序

1. 铁匠先把流星锤放到工作台
2. 再把酒杯放到工作台
3. 再把钥匙放到工作台
4. 镜头始终不给工作台特写
5. 只让玩家看到铁匠在摆放，不让玩家看清三件东西发生了什么

角色位置：

- 玩家队伍站在屏幕左侧偏后，只作为见证者
- 铁匠站在画面右侧工作台前
- 工作台只露出轮廓，不给正面特写

UI：

- 不出现任务完成面板
- 不出现统计结算
- 这一段必须像“下一轮冒险的钩子”，不是普通通关页

### 10.3 最终对白

铁匠只说一句：

> **下次回来，它们就不是现在这个样子了。**

### 10.4 收尾

1. 台词停留
2. 画面黑屏
3. Demo 结束

建议时长：

- 台词出现后停留 2 秒
- 黑屏 0.8 到 1.2 秒后结束

## 11. 当前实现前的文档结论

在真正继续实现前，先按以下理解执行：

1. 当前项目不是三关结构，而是六关结构。
2. 第一、三、五关是核心 Boss 掉落关。
3. 第二、四、六关是过渡、扩展与验证关。
4. 第二关不掉新物品。
5. 第四关的核心不是打 Boss，而是救法师、换人、给法师分配已有物品。
6. 第六关的核心不是新奖励，而是验证前面组合成果。
7. 结尾镜头不是普通结果页，而是村庄收束演出。
