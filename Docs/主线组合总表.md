# 主线组合总表

## 1. 用途

这份文档只做一件事：

- 把当前 Demo 2.0 主线中真正会出现的 `职业 × 物品` 组合整理成一张可直接查阅的表。

使用规则：

- 技能定义口径仍以 `武器组合技能.txt` 为准
- 这份表负责把主线组合整理成更容易查的二维结构
- 详细关卡演出与 UI 节奏仍看 `Demo2.0_详细设计.md`

## 4. 六关主线中实际发生的组合范围

### 4.1 第一关后

玩家会在以下组合里四选一中的前三项里做一次选择：

- 锤剑士
- 锤刺客
- 锤弓手

法师此时还未加入。

### 4.2 第三关后

玩家会在当前三人队伍基础上，再做一次“酒杯分配”。

因此此时实际可能出现：

- 基础职业 + 圣杯
- 锤系职业 + 圣杯

重点主线示例：

- 酒杯剑士
- 酒杯刺客
- 酒杯弓手
- 锤剑士再拿酒杯
- 锤刺客再拿酒杯
- 锤弓手再拿酒杯

### 4.3 第四关后

法师加入后，玩家会先换下一名旧队员，再决定是否把已有物品交给法师。

因此第四关后实际主线可出现：

- 法师
- 锤法师
- 酒法师
- 锤酒法师

备注：

- 如果实现阶段只做 `锤法师` 和 `锤酒法师`，也应在演出与流程中写清楚。

### 4.4 第五关后

钥匙加入后，主线开始进入“两两组合可读”的阶段。

此时实际会出现的，不再是“所有组合全覆盖”，而是：

- 当前仍在队伍中的三名角色
- 这三名角色各自拿到钥匙后的结果

也就是说，第五关不是要求同一次流程里把所有职业都和钥匙组合一遍，而是要求：

- 玩家当前队伍里的三个候选项要都能成立
- 旧物品路线和钥匙路线叠加后，要能读懂

---

## 5. 实现优先顺序建议

如果按最少返工方式实现，建议优先级如下：

1. 第一关锤系三职业稳定
2. 第三关酒杯系三职业稳定
3. 第四关法师、锤法师、锤酒法师
4. 第五关钥匙系四职业
5. 第五关中“旧物品 + 钥匙”的叠加展示
